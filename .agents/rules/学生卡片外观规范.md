# 学生卡片外观规范

## 核心原则

学生信息展示由 `StudentCardFace` 统一承载。座位图、右侧候选学生列表和拖拽预览必须共享姓名、学号、标签、数值属性、隐藏字段和大字模式的渲染规则。

外框允许按使用场景区分：

- 座位图使用固定比例座位卡。
- 右侧候选学生区域使用宽列表卡片，保持当前桌面端侧栏扫读效率。
- 拖拽预览复用同一学生信息展示组件，不克隆 DOM 字符串。

## 尺寸真源

卡片尺寸、圆角、边框和动效 token 定义在 `src/assets/main.css`。

### 座位卡 token

| Token | 说明 |
| --- | --- |
| `--seat-card-width` | 座位卡宽度 |
| `--seat-card-height` | 座位卡高度 |
| `--seat-card-radius` | 座位卡圆角 |
| `--seat-card-border-width` | 座位卡边框宽度 |
| `--seat-card-outer-height` | 含边框的座位卡外部高度 |

### 候选列表 token

| Token | 说明 |
| --- | --- |
| `--candidate-card-height` | 右侧候选学生宽列表行高 |
| `--candidate-card-radius` | 候选列表卡片圆角 |

## 响应式规则

座位图和普通候选网格使用 `--seat-card-*` token。右侧候选学生栏以 `display-mode="compact"` 渲染为宽列表，使用 `--candidate-card-height`。

断点变化必须优先修改 `src/assets/main.css` 中的 token，不要在 `SeatItem`、`CandidateItem` 或 `StudentList` 中重新硬编码同一尺寸。

## 实现要点

1. `SeatItem.vue` 只负责座位外壳、状态类和交互事件。
2. `CandidateItem.vue` 只负责候选学生外壳、选择、拖拽和双击行为。
3. `StudentCardFace.vue` 负责学生信息内容，不处理座位分配或候选列表业务。
4. `useDragPreview.js` 返回结构化预览数据，不返回 `contentHtml`。
5. 动画必须遵守全局 `.disable-animations` 和 `prefers-reduced-motion` 降级。

## 相关文件

- `src/components/student/StudentCardFace.vue` - 学生信息展示核心
- `src/components/seat/SeatItem.vue` - 座位图卡片外壳
- `src/components/student/CandidateItem.vue` - 候选学生卡片外壳
- `src/components/seat/SeatChart.vue` - 座位布局和拖拽预览渲染
- `src/components/student/StudentList.vue` - 候选学生列表容器
- `src/composables/useDragPreview.js` - 结构化拖拽预览状态
