# 新项目 Agent 规则迁移清单

## 初始化

- [ ] 新项目根目录创建 `AGENTS.md`。
- [ ] 创建 `.agents/project/shared-agent-guide.md` 作为公共真源。
- [ ] 创建 `.agents/features/README.md`，说明功能文档写法。
- [ ] 如果使用多个 agent 工具，建立入口模板和同步脚本。
- [ ] 在 `package.json` 或项目脚本系统中加入文档同步和漂移检查命令。

## 替换占位内容

- [ ] 替换项目名称、目标用户和项目目标。
- [ ] 替换开发、测试、类型检查、构建、部署命令。
- [ ] 替换技术栈、路由、状态管理、数据访问方式。
- [ ] 删除不适用于新项目的平台规则。
- [ ] 删除旧项目业务路径、组件名和领域模型。

## 协作边界

- [ ] 明确哪些命令不能自动运行。
- [ ] 明确哪些目录是构建产物，禁止直接修改。
- [ ] 明确临时脚本、测试文件、类型文件放置位置。
- [ ] 明确是否允许 agent 创建 Git 提交、分支、PR、Issue。
- [ ] 明确用户文档或应用内帮助的真源位置。

## 前端规范

- [ ] 记录颜色变量和主题规则。
- [ ] 记录图标库和图标导入方式。
- [ ] 记录组件命名、Composable 或 Hook 命名、常量命名。
- [ ] 记录移动端、布局高度、滚动区域等关键 UI 约束。
- [ ] 记录用户可见功能变更后的文档同步要求。

## Bug 修复流程

- [ ] 复制或改写 `workflows/bugfix-workflow.md`。
- [ ] 填入常用入口文件。
- [ ] 填入相关测试命令。
- [ ] 填入常见故障层级和项目特有陷阱。
- [ ] 明确收尾回复格式。

## 第一次验证

- [ ] 让 agent 读取新项目 `AGENTS.md` 后复述关键规则。
- [ ] 故意提出一个小修改任务，检查 agent 是否先读文件、做小 diff、运行最小验证。
- [ ] 检查入口文档和共享真源是否一致。
- [ ] 检查文档中没有旧项目名称、旧路径或不存在的命令。
