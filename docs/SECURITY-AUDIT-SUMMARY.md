## 修复优先级路线图

### 第一阶段：严重问题修复（1-2 周）

**目标**: 修复所有严重问题，消除关键安全风险

1. **禁用 GET 参数** (`common.php`)
   - 仅接受 POST 请求
   - 移除 GET 参数回退逻辑
   - 影响：所有 API 端点

2. **强制 HTTPS** (`auth.php`)
   - 生产环境启用 `REQUIRE_HTTPS = true`
   - 添加环境检测逻辑
   - 影响：所有认证操作

3. **修复 innerHTML XSS** (`main.js`)
   - 使用 `textContent` 替代 `innerHTML`
   - 创建 DOM 元素而非字符串拼接
   - 影响：错误显示

4. **移除 WebDAV 密码存储** (`useAuth.js`)
   - 不在 Cookie 中存储密码
   - 使用后端代理或每次输入
   - 影响：WebDAV 认证

5. **添加 HttpOnly Cookie** (需后端配合)
   - 后端设置 Cookie 而非前端
   - 或使用 sessionStorage + 后端会话
   - 影响：所有 Cookie

6. **全面使用 sanitizeDbKey** (`workspace.php`)
   - 所有键名操作前消毒
   - 创建辅助函数统一处理
   - 影响：所有数据库操作

7. **添加文件 ID 验证** (`workspace.php`)
   - 使用 `isValidFileId()` 验证
   - 消毒后再用作键名
   - 影响：所有文件操作

8. **限制文件大小** (`workspace.php`, `useExcelData.js`)
   - 云端工作区：5MB
   - Excel 文件：10MB
   - 影响：文件上传

9. **Token 哈希存储** (`auth.php`)
   - 存储 SHA-256 哈希而非明文
   - 验证时比对哈希值
   - 影响：所有用户会话

### 第二阶段：高危问题修复（2-3 周）

**目标**: 修复高危问题，提升整体安全性

10. **移除测试环境详细错误** (`auth.php`)

## 已验证的安全措施

项目已实施以下安全措施，值得肯定：

### 后端安全 ✅

- ✅ CSRF 保护（Double-Submit Cookie 模式）
- ✅ 密码哈希（bcrypt）
- ✅ 输入验证（用户名、文件 ID）
- ✅ 速率限制（用户名 + IP）
- ✅ 安全日志（认证事件）
- ✅ Token 验证（hash_equals 防时序攻击）

### 前端安全 ✅

- ✅ Vue 默认转义（{{ }} 插值）
- ✅ CSRF Token 生成（crypto.getRandomValues）
- ✅ Cookie 安全标志（Secure, SameSite=Lax）
- ✅ Excel 导出安全（xlsx-js-style 自动转义）

### 文件操作安全 ✅

- ✅ 路径遍历防护（path.resolve + path.relative）
- ✅ 安全响应头（X-Content-Type-Options, X-Frame-Options 等）
- ✅ 内存管理（URL.revokeObjectURL）

### 配置安全 ✅

- ✅ .gitignore 配置（排除 .env, node_modules 等）
- ✅ 环境变量使用（未硬编码 API Key）
- ✅ 路径遍历防护（server.cjs）

## 监控和响应

### 安全监控指标

1. **认证失败率**
   - 监控登录失败次数
   - 阈值：单用户 5 次/5 分钟

2. **API 错误率**
   - 监控 4xx/5xx 响应
   - 阈值：> 5%

3. **异常访问模式**
   - 监控深夜访问
   - 监控异常 IP 地址

4. **文件操作频率**
   - 监控大量文件上传/下载
   - 阈值：> 100 次/小时

### 安全事件响应流程

1. **检测** → 安全监控系统发现异常
2. **分析** → 确认是否为安全事件
3. **遏制** → 隔离受影响系统
4. **根除** → 修复漏洞
5. **恢复** → 恢复正常服务
6. **总结** → 编写事件报告
