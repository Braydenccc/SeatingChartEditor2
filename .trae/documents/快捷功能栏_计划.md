
# 快捷功能栏开发计划

## 1. 需求概述
将 StudentList.vue 中的"学生候选"标题替换为快捷功能栏，使其根据当前状态自动列出可能需要的操作。**保留右侧原有的三个按钮**。

## 2. 现状分析
- 当前 StudentList.vue 组件有固定的三个按钮：随机排位、标签设置、编辑名单（位于右侧）
- 编辑模式由 useEditMode.js 管理，包括 NORMAL、EMPTY_EDIT、SWAP、CLEAR、ZONE_EDIT 五种模式
- 左侧有"学生候选"标题和候选人数计数

## 3. 实现方案
### 3.1 修改文件
- `src/components/student/StudentList.vue`

### 3.2 功能设计
#### 3.2.1 布局结构
- 左侧：快捷功能栏（替代原"学生候选"标题）
- 右侧：保留原有的三个按钮（随机排位、标签设置、编辑名单）

#### 3.2.2 快捷功能栏显示规则
**根据不同状态显示相应的操作：**

**1. 名单为空时（students.length === 0）：**
- 显示状态提示："暂无学生数据"
- 快捷操作：
  - 导入名单（Excel）
  - 从本地打开工作区
  - 从云端打开工作区

**2. 有学生数据时：**
- 显示当前编辑模式提示
- 显示候选人数计数
- 根据编辑模式显示相应的快捷操作：
  - NORMAL 模式：提示"正常分配模式"
  - SWAP 模式：提示"交换座位模式" + 取消按钮
  - CLEAR 模式：提示"清空座位模式" + 取消按钮
  - EMPTY_EDIT 模式：提示"空置座位模式" + 取消按钮
  - ZONE_EDIT 模式：提示"选区编辑模式" + 完成按钮

### 3.3 实现步骤
1. 在 StudentList.vue 中引入 useEditMode、useWorkspace、useAuth、useExcelData
2. 创建计算属性来判断不同状态（名单是否为空、当前编辑模式等）
3. 重构模板，将左侧标题区替换为动态快捷功能栏
4. 保留右侧原有的三个按钮
5. 添加相关功能（取消编辑模式、导入名单、打开工作区等）
6. 调整样式以适应新的布局
7. 测试各种场景下的功能

## 4. 注意事项
- 遵循项目现有的代码风格和组件架构
- 保持响应式设计
- 禁止使用 Emoji
- 遵循项目的视觉规范（品牌色、阴影、圆角等）
