# 布局优化计划 - 增强版

## 现状分析

当前项目使用统一的布局配置：
- `seatConfig` 包含 `groupCount`（大组数量）、`columnsPerGroup`（每组列数）、`seatsPerColumn`（每列座位数）
- 所有大组都有完全相同的列数和行数
- `initializeSeats()` 函数使用统一的循环生成所有座位
- `organizedSeats` 按 `[group][col][row]` 组织座位数据

## 优化目标

1. 允许为每个大组配置独立的列数和行数
2. 创建可视化的座位表配置窗口
3. 在编辑 tab 中添加按钮打开配置窗口
4. 保持向后兼容性

## 新数据结构设计

### seatConfig 新结构

```javascript
const seatConfig = ref({
  groupCount: 4,
  columnsPerGroup: 2,  // 默认列数（向后兼容）
  seatsPerColumn: 7,   // 默认行数（向后兼容）
  // 新增：每大组的配置数组
  groups: [
    { columns: 2, rows: 7 },   // 第 1 组
    { columns: 2, rows: 7 },   // 第 2 组
    { columns: 2, rows: 7 },   // 第 3 组
    { columns: 2, rows: 7 }    // 第 4 组
  ],
  shiftDistance: 4
})
```

## 实施步骤

### 1. 创建可视化配置窗口组件

创建新组件：`src/components/layout/SeatConfigDialog.vue`

功能：
- 可视化预览座位表布局
- 为每个大组独立配置列数和行数
- 提供快捷操作：
  - 统一设置所有大组的列数
  - 统一设置所有大组的行数
  - 复制某个大组的配置到其他组
  - 添加/删除大组
- 实时预览布局效果
- 确认/取消按钮

### 2. 修改 useSeatChart.js

- 更新 `seatConfig` 数据结构，添加 `groups` 数组
- 修改 `initializeSeats()` 函数，根据每个大组的配置生成座位
- 添加辅助函数：
  - `getGroupConfig(groupIndex)` - 获取指定大组的配置
  - `ensureGroupsArray()` - 确保 groups 数组存在且长度匹配
- 更新 `organizedSeats` 计算属性
- 更新相关辅助函数（`isInRowRange`、`getTotalRows` 等）

### 3. 修改 SidebarPanel.vue

- 在「座位表配置」区域添加「打开配置窗口」按钮
- 保留原有的简单配置输入框作为快速设置
- 引入并使用新的 `SeatConfigDialog` 组件

### 4. 更新工作区保存/加载 (useWorkspace.js)

- 更新 `getWorkspaceJson()` 保存新的配置结构
- 更新 `migrateWorkspace()` 函数，添加从旧版本到新版本的迁移
- 确保向后兼容：旧版本工作区仍可正常加载

### 5. 兼容性检查

检查并更新以下功能：
- 智能排位 (useAssignment.js)
- 选区轮换 (useZoneRotation.js)
- 座位规则引擎 (useSeatRules.js)
- 导出功能 (useImageExport.js, useExcelData.js)
- 选区管理 (useZoneData.js)
